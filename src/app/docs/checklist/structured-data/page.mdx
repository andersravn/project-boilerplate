<Alert title="Product Schema Example">
Here's a simplified example showing how to dynamically choose between Offer and AggregateOffer based on pricing data:
</Alert>

<CodeBlock language="typescript">
{`// Product schema with dynamic offer types
const generateProductSchema = (productData, pricesData) => {
  // Process images for structured data
  const images = productData.images?.map(img => 
    \`\${img.url}?format=webp&quality=75&width=1200&height=628\`
  ) ?? [];

  // Create offers based on pricing structure
  const offers = pricesData?.length > 0
    ? pricesData.map(price => {
        const baseOffer = {
          price: price.price ?? "0",
          priceCurrency: "USD",
          availability: "InStock",
          itemCondition: "NewCondition",
          seller: {
            "@type": "Organization",
            name: "Your Store Name"
          }
        };

        // Use AggregateOffer for price ranges, Offer for fixed prices
        return price.minPrice && price.maxPrice
          ? { "@type": "AggregateOffer", ...baseOffer, 
              lowPrice: price.minPrice, 
              highPrice: price.maxPrice }
          : { "@type": "Offer", ...baseOffer };
      })
    : undefined;

  return {
    "@context": "https://schema.org",
    "@type": "Product",
    name: productData.name,
    description: productData.description,
    image: images,
    category: productData.category,
    manufacturer: {
      "@type": "Organization",
      name: "Your Brand"
    },
    ...(offers && { offers })
  };
};`}
</CodeBlock>

## Common Pitfalls to Avoid

<Alert variant="warning" title="Structured Data Best Practices">
  - Don't mark up content that's not visible to users
  - Ensure structured data matches visible page content
  - Avoid keyword stuffing in schema descriptions
  - Test all implementations thoroughly before deployment
</Alert>
