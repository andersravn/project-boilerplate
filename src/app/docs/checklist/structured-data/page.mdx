## Technical Implementation <Badge variant="default">Essential</Badge>

### JSON-LD Format
- [ ] **JSON-LD** preferred over microdata or RDFa
- [ ] **Multiple schemas** properly linked and nested
- [ ] **Context declaration** (@context: "https://schema.org")
- [ ] **Type declarations** (@type) for all entities
- [ ] **ID properties** (@id) for entity relationships
- [ ] **Graph structure** for complex relationships
- [ ] **Escape special characters** properly in JSON

### Microdata Implementation
- [ ] **Itemscope and itemtype** attributes correctly used
- [ ] **Itemprop** attributes mapped to schema properties
- [ ] **Nested microdata** structures properly implemented
- [ ] **Fallback content** visible to users
- [ ] **Mixed format handling** (JSON-LD + Microdata) if needed

### Technical Requirements
- [ ] **Valid JSON syntax** without parsing errors
- [ ] **Required properties** included for each schema type
- [ ] **Recommended properties** added for better results
- [ ] **Image URLs** absolute and accessible
- [ ] **Date formats** follow ISO 8601 standard
- [ ] **URL properties** use canonical URLs
- [ ] **Multiple languages** handled with appropriate markup

## Common Pitfalls to Avoid

<Alert variant="warning" title="Structured Data Best Practices">
  - Don't mark up content that's not visible to users
  - Ensure structured data matches visible page content
  - Avoid keyword stuffing in schema descriptions
  - Test all implementations thoroughly before deployment
</Alert>

### Technical Issues
- [ ] **Avoid hidden content** markup
- [ ] **Don't duplicate** the same schema unnecessarily
- [ ] **Ensure data accuracy** matches visible content
- [ ] **Avoid over-optimization** with excessive markup
- [ ] **Test mobile** rendering of rich results
- [ ] **Maintain consistency** across related pages
- [ ] **Update schemas** when content structure changes
