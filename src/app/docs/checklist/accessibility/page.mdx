## Semantic HTML & Structure <Badge variant="default">Essential</Badge>

### Document Structure
<ChecklistItem title="Proper HTML5 semantic elements">(header, nav, main, section, article, aside, footer)</ChecklistItem>
<ChecklistItem title="Single H1 per page">with logical heading hierarchy (H1 → H2 → H3)</ChecklistItem>

<ChecklistItem title="Page title describes">the page content and updates on navigation</ChecklistItem>
<ChecklistItem title="Language attribute">set on html element and changes declared</ChecklistItem>
<ChecklistItem title="Reading order is logical">when CSS is disabled</ChecklistItem>

### Lists and Tables
<ChecklistItem title="Proper list markup">(ul, ol, dl) for grouped content</ChecklistItem>
<ChecklistItem title="Table headers">(th) with scope attributes for data tables</ChecklistItem>
<ChecklistItem title="Table captions">or aria-label for complex tables</ChecklistItem>
<ChecklistItem title="Column/row headers">associated with data cells</ChecklistItem>
<ChecklistItem title="Layout tables avoided">(use CSS Grid/Flexbox instead)</ChecklistItem>

## Implementation Guidelines

<Alert title="Start with Foundation">
  Begin with semantic HTML and keyboard navigation. These form the foundation that other accessibility features build upon.
</Alert>

### Development Process

1. **Phase 1**: Semantic HTML, keyboard navigation, alt text
2. **Phase 2**: Color contrast, form accessibility, ARIA basics
3. **Phase 3**: Complex ARIA patterns, advanced testing, mobile optimization

### Testing Strategy

- **Continuous Testing**: Integrate automated accessibility testing in CI/CD
- **User Testing**: Include users with disabilities in your testing process
- **Expert Review**: Have accessibility experts audit your application
- **Regular Audits**: Conduct comprehensive accessibility audits quarterly

### Tools & Resources

- **Testing Tools**: axe-core, Lighthouse, WAVE, Color Oracle
- **Screen Readers**: NVDA (free), JAWS, VoiceOver, TalkBack
- **Guidelines**: WCAG 2.1 AA, WAI-ARIA Authoring Practices
- **Training**: WebAIM, Deque University, accessibility courses

<Alert variant="warning" title="Legal Compliance">
  Many jurisdictions require accessibility compliance. Check local laws like ADA (US), AODA (Ontario), EN 301 549 (EU), and DDA (Australia).
</Alert>
